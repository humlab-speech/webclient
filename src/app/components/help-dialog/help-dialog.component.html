This guide covers the main components and tools of the Visible Speech platform.

Your Projects

Each project card gives you two panels for working with your data:

Applications

Tools you can launch to interact with your project's data:

  • Annotation tool — Label, annotate, and visualize speech data using the Arctic.
  • Notebook tool — Run R or Python code in a Jupyter environment on the VISP server to analyze your data using the emuR library.
  • Arctic — A standalone transcription editor. No separate login is required, but it operates independently from the rest of the platform.

Operations

Manage your project’s workflow and resources:

  • Auto-transcribe — Submit audio files for automatic speech transcription.
  • Recording sessions — View and manage recording sessions. Each recording session corresponds to a recording event and contains its associated audio files.
  • Assign annotation tasks — Distribute annotation bundles to project members. Assignments are tightly integrated with the Transcription tool.
  • Members — Manage your project’s team members and their roles.
  • Manage recording scripts — Create and edit recording scripts. A script is a set of prompts for participants to speak during a recording session.

Data Management

The sidebar panel shows your project’s data management status. Your data is securely stored on our servers with Git for version control, providing a full history of changes.

Help & Guides

Resources available in the sidebar:

  • VISP Tutorial — Step-by-step walkthrough of the Visible Speech platform.
  • EMU Manual — In-depth documentation for the Arctic and the emuR library.
  • E-mail support — Contact the support team at support@humlab.umu.se.

Arctic

The Arctic is a rich labeling and correction tool for visualizing, annotating, and analyzing speech data. It supports multiple annotation tiers, spectrogram views, and acoustic measurements.

Using the tool

Your assigned bundles appear in the panel on the left. Only bundles assigned to you in the dashboard will be shown — even as project leader you must assign bundles to yourself before they appear here.

The main area on the right lets you play back audio, view the spectrogram, and edit annotation tiers. For a complete feature reference, see the EMU-SDMS Manual.

Use Tab / Shift+Tab to move between bundles and Space to play or stop audio.

Jupyter Notebook

You are running a Jupyter notebook session on the VISP server. Use R or Python to analyze your speech data, including the emuR library to work with your Emu-R database — no local installation needed.

Using the tool

Write and run code in the notebook editor on the right. Your project directory is visible in the file browser on the left.

Remember to save your work. You can navigate back to the dashboard and return to your session later, but idle sessions are periodically shut down — save regularly.

Code examples

  • Load your Emu-R database

    # Load the emuR library
library(emuR)

# Load your Emu-R database. Within Visible Speech, the database location is assumed to be the same for all projects,
# Within Visible Speech the database path is the same for all projects.
# This gives you a reference called 'VISP' to your speech database.
db_path <- "~/project/Data/VISP_emuDB"
VISP <- load_emuDB(db_path)

  • List sessions

    # List all recording sessions in the database
sessions <- list_sessions(VISP)
print(sessions)

  • List bundles

    # "Bundles" are individual sound files together with their associated signal files.
#”Bundles” is the name used for individual sound files and associated other signal files in a session.
# You can list all bundles in a database

list_bundles(VISP)

# You can also list all bundles or sessions matching a pattern
# In the example below you will get all the bundles of sessions with names (perhaps recording date) starting with 2024

list_bundles(VISP,sessionPattern="2024.*")

# You can always get help for a command using ¨?¨, e.g.

?list_bundles

  • Other usage

    
# Anything that is a attribute definition can be queried for labels.
# We here assume that you have a Word level set up, similar to our default setup

query(VISP,"Word !=''") #Provides a list of all non-empty word labels

query(VISP,"Word =~ '^s.*'") # Uses a regular expression to find all words beginning with ‘s’

query(VISP,"Word !=''. -> Word =~ '^s.*'") # Finds all (not empty) word labels preceeding a word starting with ‘s’  

# We can make queries that relate to the link between transcription tiers

# (note the [] braces)

query(VISP,"[Phonetic != '' ^ Word =~ '^s.*']")
